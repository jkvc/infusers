# infusers — Kickoff

**Date:** 2026-06-08

Scaffold reset and direction for custom Klein inference on fal.

## What this repo is

- **infusers/** package — platform-agnostic inferencer logic (the product)
- **apps/** — thin `fal.App` shells that load weights once and route `mode` → inferencer + hyperparams
- **jkvc** — charged API front (`withCharge`, auth) calling private fal endpoints

Hosting investigation detail also lives in mind-dump: `~/mind-dump/notes/20260607-klein-inference-hosting.md`.

## Decisions (current)

| Topic | Choice |
| --- | --- |
| GPU host | **fal** serverless (not Replicate) — custom inferencers are the product; community model catalog irrelevant |
| Public API model | **Klein 4B** distilled (Apache 2.0) |
| Demo-only model | **Klein 9B-KV** (FLUX NC license) — multi-ref editing with KV cache |
| Weight source | Hugging Face (`black-forest-labs/FLUX.2-klein-*`) |
| Standard routes | **diffusers** pipelines (`Flux2KleinPipeline`, `Flux2KleinKVPipeline`) |
| Custom algorithms | Port denoise loop from BFL [`flux2`](https://github.com/black-forest-labs/flux2) `sampling.py` / `model.py` — do not fight diffusers pipelines forever |
| Config routing | **reqm** (`~/reqm`) later — YAML-driven inferencer registry; hardcode one mode for v1 |
| Cold start | Scale to zero while building; warm pool only for live demos |

## Target shape

```
User → jkvc API (withCharge)
         → fal private app (e.g. jkvc/klein-4b)
              → setup() loads weights once
              → mode → infusers inferencer + hyperparams
```

Two deployments eventually (won't fit one GPU): Klein 4B public pool, Klein 9B-KV demo pool.
